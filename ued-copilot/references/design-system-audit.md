# 设计规范审查清单

本文档用于审查用户提供的设计规范（Design System / Design Tokens）是否完整、合理。

---

## 审查维度

### 1. 完整性检查

#### 必需的 Token 类别
- [ ] **颜色系统**
  - 品牌色（主色、辅色）
  - 中性色（灰度）
  - 语义色（成功、警告、错误、信息）
  - 背景色
  - 边框色

- [ ] **字体系统**
  - 字体家族（至少包含正文字体）
  - 字体大小体系（至少 5 个级别）
  - 字重
  - 行高

- [ ] **间距系统**
  - 基础间距单位（推荐 4 或 8）
  - 间距刻度（至少 6 个级别）

- [ ] **圆角**
  - 至少 3 个级别（无、小、大）

- [ ] **阴影**（可选，但推荐）
  - 至少 2 个级别（轻微、常规）

#### 推荐的 Token 类别
- [ ] 断点（响应式设计）
- [ ] 触控目标尺寸
- [ ] 动画时长
- [ ] Z-index 层级
- [ ] 图标尺寸

---

### 2. 合理性检查

#### 颜色系统
**品牌色**
- [ ] 是否有清晰的主色和辅色区分？
- [ ] 主色是否有色阶？（至少 3 级：浅、标准、深）
- [ ] 品牌色对比度是否达标？
  - 白底：对比度 ≥ 4.5:1
  - 深色底：对比度 ≥ 4.5:1

**中性色**
- [ ] 灰度层级是否足够？（推荐至少 5 级）
- [ ] 深色文字在白底上对比度 ≥ 4.5:1？
- [ ] 是否避免纯黑色？（推荐 #171717 而非 #000000）

**语义色**
- [ ] 成功、警告、错误色是否符合用户直觉？
  - 成功：绿色系
  - 警告：黄/橙色系
  - 错误：红色系
- [ ] 语义色对比度是否达标？

**可访问性**
- [ ] 主要文字与背景对比度 ≥ 4.5:1（WCAG AA）？
- [ ] 大文字（18px+）与背景对比度 ≥ 3:1？
- [ ] 是否考虑色盲友好？（不仅用颜色区分状态）

#### 字体系统
**字体大小**
- [ ] 移动端正文是否 ≥ 16px？（避免自动缩放）
- [ ] 桌面端正文是否 ≥ 16px？
- [ ] 字号刻度是否合理？（推荐 1.2-1.25 倍比例）
- [ ] 最小字号是否 ≥ 12px？（谨慎使用）

**行高**
- [ ] 正文行高是否 1.5-1.6？
- [ ] 标题行高是否 1.2-1.3？
- [ ] 行高是否与字号匹配？（字号越大，行高相对值可越小）

**字体家族**
- [ ] 是否使用系统字体栈？（性能优化）
- [ ] 是否考虑中文字体？（中文场景）
- [ ] 自定义字体是否有回退方案？

#### 间距系统
- [ ] 是否基于 4pt 或 8pt 网格？
- [ ] 间距刻度是否够用？（至少 6 级）
- [ ] 间距是否成倍数关系？（便于记忆和使用）
- [ ] 最小间距是否 ≥ 4px？

#### 圆角
- [ ] 圆角值是否与设计风格匹配？
  - 严肃、正式：0-4px
  - 现代、友好：8-12px
  - 俏皮、休闲：16px+
- [ ] 圆角是否有一致的层级？

#### 触控目标
- [ ] 移动端触控目标是否 ≥ 44×44px（iOS）或 48×48px（Android）？
- [ ] 桌面端鼠标目标是否 ≥ 32×32px？
- [ ] 相邻可点击元素间距是否 ≥ 8px？

#### 动画
- [ ] 动画时长是否在 100-500ms 范围？
- [ ] 是否避免过长动画？（> 700ms 影响感知速度）
- [ ] 是否定义缓动函数？（推荐 ease-out）

---

### 3. 一致性检查

#### 命名规范
- [ ] Token 命名是否一致？
  - 颜色：primary-500, gray-900
  - 间距：spacing-4, spacing-8
  - 字号：text-base, text-lg
- [ ] 是否使用语义化命名？
  - 推荐：text-base（语义）
  - 避免：text-16（具体值）

#### 数值规律
- [ ] 间距是否遵循一致的增长规律？
  - 线性：4, 8, 12, 16, 20
  - 比例：4, 8, 16, 32, 64
- [ ] 字号是否遵循一致的比例？
  - 推荐：1.2 倍或 1.25 倍
- [ ] 颜色色阶是否均匀？

---

### 4. 平台适配

#### 跨平台考虑
- [ ] 是否区分移动端和桌面端？
  - 字号
  - 间距
  - 触控目标
- [ ] 是否考虑不同 DPI？
- [ ] 是否定义响应式断点？

#### 平台特定
**iOS（如适用）**
- [ ] 触控目标是否 ≥ 44pt？
- [ ] 是否遵循 HIG 导航模式？
- [ ] 是否考虑安全区域（刘海屏）？

**Android（如适用）**
- [ ] 触控目标是否 ≥ 48dp？
- [ ] 是否使用 Material Design elevation？
- [ ] 是否定义状态颜色（pressed, focused）？

**Web（如适用）**
- [ ] 单位是否使用 px/rem/em？
- [ ] 是否考虑浏览器兼容性？
- [ ] 是否定义 hover/focus 状态？

---

### 5. 可维护性

#### 文档完整性
- [ ] 是否有使用指南？
- [ ] 是否有示例？
- [ ] Token 是否有注释说明？

#### 可扩展性
- [ ] 颜色系统是否支持深色模式？
- [ ] 是否预留扩展空间？（如 primary-50 到 primary-900）
- [ ] 是否易于添加新的语义色？

---

## 审查结果分类

### 🔴 严重问题（必须修复）
- 可访问性不达标（对比度不足）
- 触控目标过小（移动端 < 44px）
- 缺少关键 Token 类别（颜色、字体、间距）
- 数值不合理（正文字号 < 14px）

### 🟡 建议优化
- Token 体系不够完整
- 命名不够一致
- 缺少使用指南
- 未考虑响应式

### 🟢 良好实践
- 遵循 8pt 网格
- 使用语义化命名
- 考虑深色模式
- 提供完整文档

## 常见问题及建议

### 问题：品牌色对比度不足
**建议**：
- 保持品牌色用于装饰元素
- 为文字和按钮创建深色变体（对比度达标）

### 问题：间距系统混乱
**建议**：
- 统一采用 4pt 或 8pt 网格
- 使用成倍数关系：4, 8, 16, 32, 64

### 问题：字号过多
**建议**：
- 移动端：5-6 个字号足够
- 桌面端：6-8 个字号
- 使用 1.2-1.25 倍比例

### 问题：缺少深色模式
**建议**：
- 为每个颜色 token 定义深色模式变体
- 注意深色模式下的对比度要求

### 问题：触控目标尺寸不足
**建议**：
- 移动端所有按钮、链接至少 44×44px
- 相邻元素间距至少 8px
- 小图标外层添加透明 padding 扩大触控区域
