# 设计原则快速参考

本文档汇总主流设计规范的核心原则，用于快速定位问题所违反的设计标准。

---

## Apple Human Interface Guidelines (HIG)

### 三大核心原则
1. **清晰性（Clarity）**
   - 文字清晰易读，图标精确易懂
   - 功能通过装饰体现，而非妨碍

2. **遵从性（Deference）**
   - 内容优先，UI 退居其后
   - 用半透明、模糊暗示更多内容

3. **深度（Depth）**
   - 视觉层次和动效传达层次结构
   - 增强愉悦度但不失去上下文

### 关键要求
- **触控目标**：最小 44×44 点（约 7mm）
- **反馈**：每个动作都应有即时可见的反馈
- **直接操作**：用户直接操作屏幕对象
- **控件间距**：建议至少 8 点

## Google Material Design

### 核心原则
1. **Material 是隐喻**
   - 表面、边缘、阴影提供视觉线索

2. **大胆、图形化、有意图**
   - 基于印刷设计：排版、网格、空间、比例、配色

3. **动效提供意义**
   - 动效尊重用户作为主要发起者

### 关键要求
- **基准网格**：8dp
- **触控目标**：最小 48×48 dp（约 9mm）
- **文字**：4dp 基线网格
- **Elevation**：用高度（阴影）表示层级

## Nielsen 十大可用性原则

1. **系统状态可见性** - 始终让用户知道发生了什么
2. **系统与真实世界匹配** - 使用用户语言，遵循真实惯例
3. **用户控制与自由** - 提供"紧急出口"，支持撤销和重做
4. **一致性与标准** - 遵循平台惯例，相同的词语应有相同含义
5. **错误预防** - 谨慎设计以防止问题，高风险操作需二次确认
6. **识别胜于记忆** - 使对象、操作可见，用户不应记住信息
7. **灵活高效的使用** - 提供加速器，允许用户定制
8. **美观与极简设计** - 不包含无关或很少需要的信息
9. **帮助用户识别、诊断并从错误中恢复** - 用简单语言，建设性建议
10. **帮助与文档** - 最好不需要文档，如需要应易于搜索和具体

## WCAG 2.1/2.2 无障碍标准

### 四大原则（POUR）
1. **可感知（Perceivable）** - 为非文本内容提供替代，内容易于看到和听到
2. **可操作（Operable）** - 所有功能可通过键盘使用，用户有足够时间
3. **可理解（Understandable）** - 文本可读可理解，网页以可预测方式操作
4. **健壮性（Robust）** - 最大化与当前和未来工具的兼容性

### 关键合规标准
**颜色对比度**
- AA 级（最低）：
  - 正常文本（< 18px）：4.5:1
  - 大文本（≥ 18px 或 ≥ 14px 粗体）：3:1
- AAA 级（增强）：
  - 正常文本：7:1
  - 大文本：4.5:1

**触控目标尺寸**
- WCAG 2.1 AA：44×44 CSS 像素
- WCAG 2.2 AA：24×24 CSS 像素（相邻目标间有间距）

**焦点可见性**
- 所有可交互元素必须有清晰的焦点指示器
- 焦点指示器对比度至少 3:1

## ISO 9241-210 以人为中心的设计

### 六大原则
1. 设计基于对用户、任务和环境的明确理解
2. 用户贯穿设计和开发全程
3. 设计由以用户为中心的评估驱动和改进
4. 过程是迭代的
5. 设计关注整个用户体验
6. 设计团队包含多学科技能和视角

## 快速检查表

### 触控目标
- iOS: ≥ 44×44pt
- Android: ≥ 48×48dp
- Web (WCAG 2.1): ≥ 44×44px
- Web (WCAG 2.2): ≥ 24×24px (有间距)

### 对比度
- 正常文本: ≥ 4.5:1 (AA), ≥ 7:1 (AAA)
- 大文本: ≥ 3:1 (AA), ≥ 4.5:1 (AAA)
- UI 组件: ≥ 3:1

### 字体大小
- 移动端正文: ≥ 16px (避免 iOS 自动缩放)
- 桌面端正文: ≥ 16px
- 最小文本: ≥ 12px (谨慎使用)

### 间距
- 推荐: 8pt 网格系统
- 最小间距: 4px
- 触控元素间距: ≥ 8px

### 行高
- 正文: 1.5-1.6
- 标题: 1.2-1.3

## 平台特定注意事项

### iOS
- 使用标准手势（侧滑返回）
- 遵循导航栏规范
- 适配安全区域（刘海屏）
- 支持深色模式

### Android
- 使用 Material Design elevation
- FAB 放置在右下角
- 遵循 Material 过渡动效
- 使用 Material Icons

### Web
- 响应式设计
- 支持键盘操作（Tab 顺序）
- 链接有 hover 和 focus 状态
- 避免在移动端使用 hover

## 常见违规示例

### 触控目标过小
❌ 移动端按钮 32×32px
✅ 调整为 44×44px

### 对比度不足
❌ #999 灰色文字在白底上 (2.8:1)
✅ 使用 #595959 (7:1) 或 #767676 (4.6:1)

### 缺少反馈
❌ 点击按钮无任何响应
✅ 添加加载态、禁用状态、成功提示

### 仅用颜色传达信息
❌ 只用红色表示必填字段
✅ 红色 + "*" 符号 + aria-required

### 不可逆操作无确认
❌ 点击删除按钮立即删除
✅ 弹窗确认："确定删除此项目吗？"
