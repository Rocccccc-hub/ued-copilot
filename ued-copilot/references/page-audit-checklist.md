# 页面 UI/UX 审查清单

本文档用于系统化审查页面设计，快速识别 UI 和 UX 问题。

---

## 审查流程

1. **整体印象**（10 秒扫视）
2. **视觉层面**（UI）
3. **交互层面**（UX）
4. **可用性原则**（Nielsen）
5. **无障碍性**（WCAG）
6. **平台规范**（HIG/Material）

---

## 1. 整体印象评估

**快速扫描问题**：
- 页面目的是否一目了然？
- 视觉焦点是否清晰？
- 信息密度是否合理？（过载 / 过疏）
- 第一眼是否知道该做什么？

---

## 2. 视觉层面（UI）

### 2.1 排版与可读性

**字体大小**
- [ ] 移动端正文 ≥ 16px？
- [ ] 桌面端正文 ≥ 16px？
- [ ] 辅助文字 ≥ 12px？（最小值）
- [ ] 按钮文字 ≥ 14px？

**行高与字间距**
- [ ] 正文行高 1.5-1.6？
- [ ] 标题行高 1.2-1.3？
- [ ] 字间距是否舒适？

**字重对比**
- [ ] 标题是否比正文更粗？
- [ ] 强调文字是否有字重区分？

**可读性**
- [ ] 每行文字长度是否合理？（移动端 < 75 字符，桌面端 < 90 字符）
- [ ] 段落间距是否足够？
- [ ] 是否避免全大写正文？

**常见问题**：
- ❌ 移动端正文 14px（iOS 会自动放大）
- ❌ 行高 1.2 的大段正文（难以阅读）
- ❌ 浅灰色小字（对比度不足）

---

### 2.2 颜色与对比

**对比度检查**
- [ ] 主要文字对比度 ≥ 4.5:1？
- [ ] 辅助文字对比度 ≥ 4.5:1？（或 ≥ 3:1 如果是大文字）
- [ ] 链接颜色是否区别于正文？
- [ ] 按钮文字在按钮背景上对比度 ≥ 4.5:1？

**颜色语义**
- [ ] 成功、警告、错误是否使用标准颜色？
  - 成功：绿色
  - 警告：黄/橙色
  - 错误：红色
- [ ] 是否仅依赖颜色传达信息？（色盲问题）

**配色一致性**
- [ ] 品牌色使用是否一致？
- [ ] 背景色是否层次分明？

**常见问题**：
- ❌ 灰色文字在浅灰背景上（对比度不足）
- ❌ 蓝色按钮 + 蓝色文本链接（无法区分）
- ❌ 仅用颜色表示必填字段

---

### 2.3 间距与对齐

**间距一致性**
- [ ] 组件内边距是否一致？
- [ ] 组件间距是否遵循网格？（推荐 8pt）
- [ ] 页面边距是否合理？
  - 移动端：16-20px
  - 桌面端：24-32px
- [ ] 内容是否紧贴边缘？（需要内边距）

**对齐**
- [ ] 文字对齐是否一致？
- [ ] 图标与文字是否垂直居中？
- [ ] 表单标签与输入框是否对齐？

**分组**
- [ ] 相关元素是否靠近？（格式塔原理）
- [ ] 不相关元素是否有足够间距？
- [ ] 章节分隔是否明确？

**常见问题**：
- ❌ 按钮文字紧贴边缘（缺少内边距）
- ❌ 相关信息分散（用户需要视觉跳跃）
- ❌ 间距混乱（8px、10px、12px、15px 混用）

---

### 2.4 信息层级

**视觉权重**
- [ ] 主标题是否最突出？
- [ ] 次要信息是否弱化？（颜色、大小）
- [ ] 重要操作是否突出？（按钮样式）

**层级工具**
- [ ] 大小对比是否明显？
- [ ] 颜色对比是否合理？
- [ ] 字重对比是否清晰？
- [ ] 是否有过多层级？（建议 ≤ 3 级）

**常见问题**：
- ❌ 所有文字同样大小、同样颜色
- ❌ 次要操作比主操作更突出
- ❌ 标题与正文区分不明显

---

### 2.5 组件一致性

**按钮**
- [ ] 主按钮样式唯一？（Primary）
- [ ] 次按钮与主按钮可区分？
- [ ] 文本按钮是否轻量？
- [ ] 危险操作是否有警示色？
- [ ] 按钮高度是否一致？
  - 移动端：≥ 44px
  - 桌面端：≥ 32px

**输入框**
- [ ] 默认、焦点、禁用、错误状态是否清晰？
- [ ] 输入框高度是否一致？
- [ ] Label 与输入框关联是否明确？
- [ ] Placeholder 是否提供示例？

**卡片/容器**
- [ ] 圆角是否一致？
- [ ] 阴影是否统一？
- [ ] 内边距是否一致？

**图标**
- [ ] 图标风格是否统一？（线性 / 面性）
- [ ] 图标大小是否一致？
- [ ] 图标与文字是否垂直对齐？

**常见问题**：
- ❌ 页面内有多个"主按钮"
- ❌ 图标混用（有的线性、有的面性）
- ❌ 卡片圆角不一致（4px、8px 混用）

---

### 2.6 响应式设计

- [ ] 移动端布局是否适配？
- [ ] 是否避免横向滚动？
- [ ] 关键内容是否在首屏？
- [ ] 触控目标是否足够大？
- [ ] 导航在移动端是否可用？（汉堡菜单 / 底部导航）

**常见问题**：
- ❌ 桌面端设计直接缩放到移动端
- ❌ 移动端触控目标 < 44px
- ❌ 移动端表格横向滚动

---

## 3. 交互层面（UX）

### 3.1 导航与信息架构

**导航清晰性**
- [ ] 用户知道自己在哪里？（当前页高亮）
- [ ] 返回路径是否清晰？
- [ ] 菜单层级是否合理？（≤ 3 层）
- [ ] 搜索功能是否易找到？

**面包屑**
- [ ] 深层级页面是否有面包屑？
- [ ] 面包屑是否可点击？

**常见问题**：
- ❌ 导航没有当前页标识
- ❌ 移动端深层页面无返回按钮
- ❌ 菜单层级过深（需要多次点击）

---

### 3.2 任务流程

**路径优化**
- [ ] 关键任务路径是否最短？
- [ ] 是否有不必要的步骤？
- [ ] 多步骤流程是否有进度指示？
- [ ] 是否支持跳过非必要步骤？

**表单设计**
- [ ] 字段顺序是否符合逻辑？
- [ ] 是否只要求必要信息？
- [ ] 是否支持自动填充？
- [ ] 长表单是否分页/分组?

**常见问题**：
- ❌ 注册需要填写过多信息
- ❌ 多步表单没有进度条
- ❌ 无法保存草稿（刷新后丢失）

---

### 3.3 反馈与状态

**加载状态**
- [ ] 按钮点击是否有加载态？
- [ ] 页面加载是否有指示器？（骨架屏 / Spinner）
- [ ] 长时间操作是否有进度条？

**操作反馈**
- [ ] 操作成功是否有提示？
- [ ] 操作失败是否有提示？
- [ ] 反馈是否及时？（< 1 秒）

**系统状态**
- [ ] 在线/离线状态是否可见？
- [ ] 同步状态是否显示？
- [ ] 网络错误是否提示？

**常见问题**：
- ❌ 按钮点击无任何反馈
- ❌ 操作成功后无提示（用户不确定是否成功）
- ❌ 加载时间长但无进度指示

---

### 3.4 错误处理

**错误提示**
- [ ] 错误提示是否用用户语言？（非技术术语）
- [ ] 是否准确指出问题？
- [ ] 是否提供解决方案？
- [ ] 表单错误是否在字段附近显示？

**预防性设计**
- [ ] 是否有输入格式提示？
- [ ] 是否有输入限制？（字符数、格式）
- [ ] 是否有实时验证？

**错误恢复**
- [ ] 是否支持快速重试？
- [ ] 是否保留用户已输入数据？
- [ ] 是否提供替代方案？

**常见问题**：
- ❌ "错误代码：500"（用户看不懂）
- ❌ 表单提交失败，已填数据丢失
- ❌ 错误提示在页面顶部，字段在底部（需要滚动才能看到）

---

### 3.5 操作确认

**不可逆操作**
- [ ] 删除是否有二次确认？
- [ ] 提交是否有确认？
- [ ] 确认弹窗文案是否明确？
  - ❌ "确定" / "取消"
  - ✅ "删除" / "保留"

**危险操作**
- [ ] 是否有视觉警示？（红色）
- [ ] 是否解释后果？
- [ ] 批量操作是否有预览？

**常见问题**：
- ❌ 删除按钮无确认，误点即删除
- ❌ 确认弹窗文案模糊："确定要继续吗？"
- ❌ 危险操作与普通操作视觉无区别

---

### 3.6 内容与文案

**按钮文案**
- [ ] 是否行动导向？
  - ✅ "保存更改"
  - ❌ "确定"
- [ ] 是否清晰明确？
- [ ] 是否简洁？（移动端尤其重要）

**空状态**
- [ ] 首次使用是否有引导？
- [ ] 无数据时是否有提示？
- [ ] 是否提供行动建议？

**帮助文档**
- [ ] 复杂功能是否有说明？
- [ ] 帮助信息是否易于访问？（? 图标）
- [ ] 工具提示是否简洁？

**常见问题**：
- ❌ "确定" / "取消" 按钮文案（不够明确）
- ❌ 空列表无任何提示
- ❌ 占位符无示例

---

## 4. 可用性原则（Nielsen）

### 4.1 系统状态可见性
- [ ] 用户是否始终知道系统在做什么？
- [ ] 加载、保存、同步是否有提示？

### 4.2 系统与真实世界匹配
- [ ] 是否使用用户熟悉的语言？
- [ ] 信息顺序是否符合逻辑？

### 4.3 用户控制与自由
- [ ] 是否支持撤销？
- [ ] 是否有明确的退出路径？
- [ ] 模态弹窗是否易于关闭？

### 4.4 一致性与标准
- [ ] 相同操作是否使用相同词汇？
- [ ] 是否遵循平台惯例？

### 4.5 错误预防
- [ ] 是否有输入验证？
- [ ] 危险操作是否难以误触？

### 4.6 识别胜于记忆
- [ ] 选项是否可见？
- [ ] 是否提供历史记录/建议？

### 4.7 灵活高效使用
- [ ] 是否有键盘快捷键？
- [ ] 是否支持批量操作？

### 4.8 美观与极简
- [ ] 是否移除了不必要的信息？
- [ ] 每个元素是否都有目的？

### 4.9 错误识别与恢复
- [ ] 错误提示是否清晰？
- [ ] 是否提供纠正建议？

### 4.10 帮助与文档
- [ ] 核心功能是否无需文档即可理解？
- [ ] 复杂功能是否有上下文帮助？

---

## 5. 无障碍性（WCAG）

### 5.1 颜色与对比
- [ ] 文字对比度 ≥ 4.5:1？（AA）
- [ ] 大文字对比度 ≥ 3:1？
- [ ] 是否仅用颜色传达信息？

### 5.2 键盘操作
- [ ] 所有功能是否可用键盘完成？
- [ ] Tab 顺序是否合理？
- [ ] 焦点指示器是否清晰？

### 5.3 触控目标
- [ ] 移动端触控目标 ≥ 44×44px（iOS）？
- [ ] Android ≥ 48×48dp？
- [ ] 相邻元素间距 ≥ 8px？

### 5.4 语义化
- [ ] HTML 语义是否正确？
- [ ] 图片是否有 alt？
- [ ] 表单是否有 label？
- [ ] 按钮是否用 button 标签？

### 5.5 屏幕阅读器
- [ ] ARIA 标签是否正确？
- [ ] 图标是否有 aria-label？
- [ ] 动态内容是否有 aria-live？

### 5.6 媒体与时间
- [ ] 视频是否有字幕？
- [ ] 自动播放是否可控？
- [ ] 闪烁内容是否 < 3 次/秒？

---

## 6. 平台规范

### iOS (Apple HIG)
- [ ] 触控目标 ≥ 44×44pt？
- [ ] 是否使用标准手势？（侧滑返回）
- [ ] 导航栏是否遵循标准？
- [ ] 是否适配深色模式？
- [ ] 是否考虑安全区域？（刘海屏）

### Android (Material Design)
- [ ] 触控目标 ≥ 48×48dp？
- [ ] 是否使用 elevation（阴影）表示层级？
- [ ] FAB 使用是否合理？
- [ ] 是否使用 Material 动效？

### Web
- [ ] 是否响应式？
- [ ] 是否适配常见浏览器？
- [ ] 链接是否有 :hover 和 :focus？
- [ ] 是否避免移动端使用 hover？

## 审查输出格式

```markdown
## UI/UX 审查报告

### 页面整体评估
- **可用性评分**：75/100
- **主要风险**：中等
- **优先修复**：
  1. 移动端触控目标过小
  2. 表单错误提示位置不合理
  3. 删除操作缺少确认

### 问题清单

#### 1. 移动端按钮触控目标过小
- **类型**：UI / 无障碍
- **违反原则**：Apple HIG, WCAG 2.1
- **问题说明**：主要操作按钮高度仅 32px，低于 iOS 最小触控目标 44pt
- **用户影响**：用户难以准确点击，尤其在移动场景下
- **修复建议**：
  - 按钮高度调整为 44px
  - 内边距调整为上下 12px，左右 24px
  - 确保相邻按钮间距至少 8px
- **参考**：Apple HIG - Buttons

#### 2. ...

### 整体优化建议
- **信息架构**：建议简化导航层级，将当前 4 层减少到 3 层
- **交互流程**：注册流程可减少 2 个步骤，提升转化率
- **视觉一致性**：建议统一圆角值，当前混用了 4px、6px、8px

### 进阶建议
- 考虑引入设计系统，统一组件样式
- 关键页面建议 A/B 测试按钮文案
```

## 常见模式识别

### 反模式 1：视觉噪音过多
**表现**：多种颜色、多种字体、过多装饰
**影响**：分散注意力，信息层级混乱
**修复**：简化配色（2-3 种主色），统一字体，移除装饰元素

### 反模式 2：操作反馈缺失
**表现**：点击按钮无反应，操作成功无提示
**影响**：用户不确定操作是否成功，可能重复操作
**修复**：添加加载态、成功提示、错误提示

### 反模式 3：表单设计不当
**表现**：字段过多、标签不清晰、错误提示不明确
**影响**：用户填写困难，放弃率高
**修复**：减少字段、优化标签、实时验证、清晰错误提示

### 反模式 4：移动端不友好
**表现**：触控目标小、文字小、横向滚动
**影响**：移动端体验差，用户流失
**修复**：增大触控目标、调整字号、优化布局

### 反模式 5：缺少空状态
**表现**：无数据时显示空白页面
**影响**：用户困惑，不知道下一步做什么
**修复**：添加引导文案和操作建议
